Once your virtual machines are running, VirtCloud gives you full control over their lifecycle — from routine power operations like starting and rebooting, to configuration changes like renaming and tagging. This guide covers every management action available through the dashboard and REST API.

## VM Status Overview

At any point in time, a VM is in exactly one of the following states:

| Status | Description |
|--------|-------------|
| `provisioning` | The VM is being created and configured for the first time. |
| `running` | The VM is online and accepting connections. |
| `stopping` | A stop action has been requested; the VM is gracefully shutting down. |
| `stopped` | The VM is powered off. You are not billed for compute while stopped, but storage billing continues. |
| `rebooting` | The VM is restarting; it will return to `running` automatically. |
| `error` | An unexpected failure occurred. Contact support with the VM ID for assistance. |

## Power Actions

<Tabs>
  <Tab title="Dashboard">
    <Steps>
      <Step title="Navigate to your VM">
        In the left sidebar, click **Virtual Machines** and select the VM you want to manage from the list.
      </Step>
      <Step title="Open the Actions menu">
        On the VM detail page, click the **Actions** dropdown in the top-right corner.
      </Step>
      <Step title="Select a power action">
        Choose **Start**, **Stop**, or **Reboot** from the menu. A confirmation dialog will appear for Stop and Reboot actions. Confirm to proceed.
      </Step>
    </Steps>
  </Tab>
  <Tab title="API">
    All power actions follow the same pattern: send a `POST` request to the action endpoint for your VM.

    **Start a stopped VM**

    <CodeGroup>
      ```bash cURL
      curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4/start \
        -H "Authorization: Bearer YOUR_API_KEY"
      ```
    </CodeGroup>

    **Stop a running VM**

    <CodeGroup>
      ```bash cURL
      curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4/stop \
        -H "Authorization: Bearer YOUR_API_KEY"
      ```
    </CodeGroup>

    **Reboot a running VM**

    <CodeGroup>
      ```bash cURL
      curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4/reboot \
        -H "Authorization: Bearer YOUR_API_KEY"
      ```
    </CodeGroup>

    All three endpoints return `200 OK` with the updated VM object reflecting the new `status`.
  </Tab>
</Tabs>

## Viewing VM Details

Retrieve the full detail record for a single VM using its ID:

<CodeGroup>
  ```bash cURL
  curl https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4 \
    -H "Authorization: Bearer YOUR_API_KEY"
  ```
</CodeGroup>

<CodeGroup>

## Renaming and Tagging

Update a VM's name or tags at any time with a `PUT` request. The VM does not need to be stopped for this operation.

<CodeGroup>
  ```bash cURL
  curl -X PUT https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4 \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "web-server-primary",
      "tags": ["production", "web", "nginx"]
    }'
  ```
</CodeGroup>

The response returns the updated VM object. Tag updates replace the full tag array — include all tags you want to keep, not just the new ones.

## Resizing a VM

You can change a VM's size to scale compute resources up or down using the dashboard. VirtCloud uses **cold resize**, which means the VM must be fully stopped before the resize can be applied.

<Warning>
  You must stop your VM before resizing it. Make sure to gracefully stop any running workloads before proceeding.
</Warning>

<Steps>
  <Step title="Stop the VM">
    Use the **Actions** menu on the VM detail page to stop the VM (see [Power Actions](#power-actions) above), then wait until the status transitions to `stopped`.
  </Step>
  <Step title="Open Resize settings">
    On the VM detail page, click **Actions** and select **Resize VM**. A dialog will appear showing the available sizes.
  </Step>
  <Step title="Select a new size">
    Choose one of the available sizes — `c1.small`, `c1.medium`, `c2.large`, or `c2.xlarge` — and click **Apply Resize**.
  </Step>
  <Step title="Restart the VM">
    After the resize completes and the VM status returns to `stopped`, start the VM again to bring it back online with its new hardware allocation.
  </Step>
</Steps>

<Note>
  Root disk size is tied to the VM size and increases automatically when you upgrade. Resizing to a smaller size does not reduce the root disk — you will retain the larger disk from any previous upgrade.
</Note>

## Accessing the Console

If you lose SSH access or need to troubleshoot a VM at the OS level, use the **VNC console** in the dashboard for direct headless access.

<Steps>
  <Step title="Open the VM detail page">
    Navigate to **Virtual Machines** in the sidebar and select your VM.
  </Step>
  <Step title="Launch the console">
    Click the **Console** button near the top of the VM detail page. A new browser tab will open with a live VNC session into the VM's display output.
  </Step>
</Steps>

The VNC console is available regardless of the VM's network configuration and does not require an open SSH port.

## Deleting a VM

<Warning>
  Deleting a VM is **permanent and irreversible**. All data on the root disk is destroyed immediately. Any attached block volumes are detached but not deleted — you can find them in the **Volumes** section. Create a [snapshot](/vms/snapshots) before deleting if you need to preserve the disk state.
</Warning>

<Tabs>
  <Tab title="Dashboard">
    <Steps>
      <Step title="Navigate to the VM">
        Go to **Virtual Machines** and select the VM you want to delete.
      </Step>
      <Step title="Open Actions and select Delete">
        Click the **Actions** dropdown and select **Delete VM**.
      </Step>
      <Step title="Confirm deletion">
        Type the VM name into the confirmation field and click **Delete** to confirm. The VM will be removed immediately.
      </Step>
    </Steps>
  </Tab>
  <Tab title="API">
    <CodeGroup>
      ```bash cURL
      curl -X DELETE https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4 \
        -H "Authorization: Bearer YOUR_API_KEY"
      ```
    </CodeGroup>

    A successful deletion returns `204 No Content` with an empty body.
  </Tab>
</Tabs>
