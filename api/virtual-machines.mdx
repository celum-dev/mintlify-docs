Virtual machines are the core compute resource in VirtCloud. You can create, configure, power-cycle, resize, and delete VMs entirely through the REST API. All endpoints are authenticated with a Bearer token and return JSON. The base URL for every request is `https://api.virtcloud.io/v1`.

---

## List virtual machines

<ParamField query="page" type="integer" default="1">
  The page number to return.
</ParamField>

<ParamField query="per_page" type="integer" default="20">
  Number of results per page. Maximum is 200.
</ParamField>

<ParamField query="region" type="string">
  Filter results to a specific region slug, for example `us-east-1`.
</ParamField>

<ParamField query="status" type="string">
  Filter by VM status. Accepted values: `running`, `stopped`, `starting`, `stopping`, `rebooting`, `error`.
</ParamField>

<CodeGroup>
```bash cURL
curl https://api.virtcloud.io/v1/virtual-machines \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash cURL (filtered)
curl "https://api.virtcloud.io/v1/virtual-machines?region=us-east-1&status=running&per_page=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

**`GET /virtual-machines`** — Returns `200 OK`

## Create a virtual machine

**`POST /virtual-machines`** — Returns `201 Created`

You must supply exactly one boot source: `image`, `snapshot_id`, `template_id`, or `backup_id`.

<ParamField body="name" type="string" required>
  A unique, human-readable name for the VM. Must be between 1 and 128 characters.
</ParamField>

<ParamField body="region" type="string" required>
  The region slug where the VM will be created, for example `us-east-1`.
</ParamField>

<ParamField body="size" type="string" required>
  The size slug that defines the VM's vCPU count, memory, and root disk, for example `c1.medium` or `c2.large`.
</ParamField>

<ParamField body="image" type="string">
  An OS image slug to boot from, for example `ubuntu-22.04`. Required when `snapshot_id`, `template_id`, and `backup_id` are all absent.
</ParamField>

<ParamField body="snapshot_id" type="string">
  ID of a snapshot to restore as the boot disk. Mutually exclusive with `image`, `template_id`, and `backup_id`.
</ParamField>

<ParamField body="template_id" type="string">
  ID of a VM template to provision from. Mutually exclusive with `image`, `snapshot_id`, and `backup_id`.
</ParamField>

<ParamField body="backup_id" type="string">
  ID of a backup to restore. Mutually exclusive with `image`, `snapshot_id`, and `template_id`.
</ParamField>

<ParamField body="ssh_key_ids" type="array">
  List of SSH key IDs to inject into the VM at boot. Keys must already exist in your account.
</ParamField>

<ParamField body="vpc_id" type="string">
  ID of the VPC to attach the VM to. If omitted, the VM is placed in your account's default VPC for the region.
</ParamField>

<ParamField body="tags" type="array">
  Arbitrary string tags to attach to the VM for grouping and filtering.
</ParamField>

<ParamField body="user_data" type="string">
  A Base64-encoded cloud-init script executed on first boot. Maximum 64 KB when decoded.
</ParamField>

<ParamField body="volumes" type="array">
  List of existing volume IDs to attach to the VM at creation time.
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST https://api.virtcloud.io/v1/virtual-machines \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "web-server-01",
    "region": "us-east-1",
    "size": "c1.medium",
    "image": "ubuntu-22.04",
    "ssh_key_ids": ["key_a1b2c3"],
    "tags": ["production", "web"]
  }'
```

```bash cURL (from snapshot)
curl -X POST https://api.virtcloud.io/v1/virtual-machines \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "restored-server",
    "region": "us-east-1",
    "size": "c1.medium",
    "snapshot_id": "snap_abc123"
  }'
```
</CodeGroup>

```json Response (201 Created)
{
  "data": {
    "id": "vm_a1b2c3d4",
    "name": "web-server-01",
    "status": "starting",
    "region": "us-east-1",
    "size": "c1.medium",
    "image": "ubuntu-22.04",
    "public_ip": "203.0.113.10",
    "private_ip": "10.0.1.5",
    "ssh_key_ids": ["key_a1b2c3"],
    "vpc_id": "vpc_x9y8z7",
    "tags": ["production", "web"],
    "user_data": null,
    "volumes": [],
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

## Update a virtual machine

**`PATCH /virtual-machines/{id}`** — Returns `200 OK`

Update the name or tags of an existing virtual machine. You can apply this change regardless of the VM's current power state.

<ParamField path="id" type="string" required>
  The ID of the virtual machine to update.
</ParamField>

<ParamField body="name" type="string">
  A new name for the virtual machine.
</ParamField>

<ParamField body="tags" type="array">
  Replaces the VM's tag list entirely with the provided array. Pass an empty array to remove all tags.
</ParamField>

<CodeGroup>
```bash cURL
curl -X PATCH https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "web-server-primary", "tags": ["production", "web", "primary"]}'
```
</CodeGroup>

```json Response (200 OK)
{
  "data": {
    "id": "vm_a1b2c3d4",
    "name": "web-server-primary",
    "status": "running",
    "region": "us-east-1",
    "size": "c1.medium",
    "image": "ubuntu-22.04",
    "public_ip": "203.0.113.10",
    "private_ip": "10.0.1.5",
    "tags": ["production", "web", "primary"],
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

## Replace a virtual machine's configuration

**`PUT /virtual-machines/{id}`** — Returns `200 OK`

Replaces the mutable configuration of a virtual machine in a single atomic operation. Unlike `PATCH`, which applies partial changes, `PUT` sets the `name`, `tags`, and `user_data` fields to exactly the values you provide. Any field you omit is reset to its default value, so you must include all desired values in every request.

<ParamField path="id" type="string" required>
  The ID of the virtual machine to replace.
</ParamField>

<ParamField body="name" type="string" required>
  The new name for the virtual machine. Must be between 1 and 128 characters.
</ParamField>

<ParamField body="tags" type="array" required>
  The complete list of string tags for the VM. Pass an empty array to clear all tags.
</ParamField>

<ParamField body="user_data" type="string">
  A Base64-encoded cloud-init script. Pass `null` to clear any existing user data. Changes to this field take effect on the next boot.
</ParamField>

<CodeGroup>
```bash cURL
curl -X PUT https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4 \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "web-server-primary",
    "tags": ["production", "web"],
    "user_data": null
  }'
```
</CodeGroup>

```json Response (200 OK)
{
  "data": {
    "id": "vm_a1b2c3d4",
    "name": "web-server-primary",
    "status": "running",
    "region": "us-east-1",
    "size": "c1.medium",
    "image": "ubuntu-22.04",

}
```

## Delete a virtual machine

**`DELETE /virtual-machines/{id}`** — Returns `204 No Content`

Permanently deletes a virtual machine and releases its IP address. Attached volumes are detached but **not** deleted. The operation is irreversible.

<ParamField path="id" type="string" required>
  The ID of the virtual machine to delete.
</ParamField>

<Warning>
  Deleting a VM is permanent. All data on the VM's root disk is destroyed. Any attached volumes are detached and preserved in your account, but you should verify this before proceeding if the volumes contain important data.
</Warning>

<CodeGroup>
```bash cURL
curl -X DELETE https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4 \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

A successful request returns `204 No Content` with no response body.

## Stop a virtual machine

**`POST /virtual-machines/{id}/stop`** — Returns `200 OK`

Gracefully shuts down a running virtual machine by sending an ACPI power-off signal to the guest OS. The VM transitions to `stopping` and then `stopped`.

<ParamField path="id" type="string" required>
  The ID of the virtual machine to stop.
</ParamField>

<Note>
  This endpoint triggers a graceful shutdown. If your VM's OS does not respond to the ACPI signal within 90 seconds, the VM is forcibly powered off to prevent it from remaining in a `stopping` state indefinitely.
</Note>

<CodeGroup>
```bash cURL
curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4/stop \
  -H "Authorization: Bearer YOUR_API_KEY"
```
</CodeGroup>

```json Response (200 OK)
{
  "data": {
    "id": "vm_a1b2c3d4",
    "name": "web-server-primary",
    "status": "stopping",
    "region": "us-east-1",
    "size": "c1.medium",
    "public_ip": "203.0.113.10",
    "private_ip": "10.0.1.5",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

## Resize a virtual machine

**`POST /virtual-machines/{id}/resize`** — Returns `200 OK`

Changes the size (vCPU and RAM tier) of a virtual machine. You can only resize a VM while it is in the `stopped` state.

<ParamField path="id" type="string" required>
  The ID of the virtual machine to resize.
</ParamField>

<ParamField body="size" type="string" required>
  The target size slug, for example `c2.large`. The new size must be available in the VM's current region.
</ParamField>

<Warning>
  The VM must be **stopped** before you call this endpoint. Attempting to resize a running or transitioning VM returns a `409 Conflict` error. Stop the VM first using `POST /virtual-machines/{id}/stop`, then issue the resize request.
</Warning>

<CodeGroup>
```bash cURL
curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4/resize \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"size": "c2.large"}'
```
</CodeGroup>

```json Response (200 OK)
{
  "data": {
    "id": "vm_a1b2c3d4",
    "name": "web-server-primary",
    "status": "stopped",
    "region": "us-east-1",
    "size": "c2.large",
    "public_ip": "203.0.113.10",
    "private_ip": "10.0.1.5",
    "created_at": "2024-01-15T10:30:00Z"
  }
}
```

<Tip>
  After a successful resize, start the VM again with `POST /virtual-machines/{id}/start`. The new vCPU and memory allocation will be active immediately on boot.
</Tip>

## Add an SSH key to a virtual machine

**`POST /virtual-machines/{id}/ssh-keys`** — Returns `200 OK`

Associates an existing SSH key from your account with the virtual machine. The key is added to the `authorized_keys` file of the default user on the next boot, or immediately if the VirtCloud guest agent is running.

<ParamField path="id" type="string" required>
  The ID of the virtual machine.
</ParamField>

<ParamField body="ssh_key_id" type="string" required>
  The ID of the SSH key to attach. The key must already exist in your VirtCloud account.
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_a1b2c3d4/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"ssh_key_id": "key_d4e5f6"}'
```
