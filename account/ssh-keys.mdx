SSH keys are the primary authentication method for connecting to VirtCloud virtual machines. By default, VirtCloud disables password-based SSH login on all new VMs — you must use a registered SSH key to gain shell access. Managing your keys through VirtCloud lets you inject the right public key at VM creation time without any post-boot configuration.

## Generating an SSH key pair

If you don't already have an SSH key pair, generate one on your local machine before proceeding.

```bash
ssh-keygen -t ed25519 -C "your-email@example.com"
# Follow the prompts to choose a file location and optional passphrase.
# Your public key is saved at ~/.ssh/id_ed25519.pub

# Print your public key so you can copy it:
cat ~/.ssh/id_ed25519.pub
```

<Tip>
  The `ed25519` algorithm produces shorter, faster, and more secure keys than the older `rsa` type. Use it whenever your toolchain supports it.
</Tip>

<Warning>
  Never share your **private key** (`~/.ssh/id_ed25519`) with anyone, upload it to any service, or store it in version control. Only your **public key** (ending in `.pub`) is meant to be shared.
</Warning>

## Adding your public key to VirtCloud

<Steps>
  <Step title="Open SSH Key Settings">
    In the VirtCloud dashboard, navigate to **Settings → SSH Keys**.
  </Step>
  <Step title="Add a new key">
    Click **Add SSH Key**, paste the contents of your `.pub` file into the **Public Key** field, and give the key a memorable name (for example, `work-laptop` or `home-desktop`).
  </Step>
  <Step title="Save the key">
    Click **Save**. The key is now available to attach to VMs.
  </Step>
</Steps>

You can also add keys programmatically using the API:

<CodeGroup>
```bash cURL
curl -X POST https://api.virtcloud.io/v1/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-macbook",
    "public_key": "ssh-ed25519 AAAA...your-public-key-here your-email@example.com"
  }'
```

```json Response
{
  "id": "sk_abc123",
  "name": "my-macbook",
  "fingerprint": "SHA256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "created_at": "2024-11-01T10:30:00Z"
}
```
</CodeGroup>

Save the returned `id` — you'll use it to attach the key when creating VMs.

## Using an SSH key when creating a VM

When making a VM create request, pass one or more SSH key IDs in the `ssh_key_ids` array:

```json
{
  "name": "web-server-01",
  "region": "us-east-1",
  "size": "c1.medium",
  "image": "ubuntu-22.04",
  "ssh_key_ids": ["sk_abc123"]
}
```

VirtCloud injects the corresponding public key into the VM's `~/.ssh/authorized_keys` during provisioning. You can specify multiple key IDs to allow access from more than one device.

## Connecting to a VM

Once your VM is running, connect using the private key that corresponds to the public key you registered:

```bash
# Specify the key explicitly:
ssh -i ~/.ssh/id_ed25519 root@<vm-public-ip>

# Or, if id_ed25519 is your default SSH key, the SSH agent picks it up automatically:
ssh root@<vm-public-ip>
```

Replace `<vm-public-ip>` with the IP address shown on your VM's detail page in the dashboard.

## Listing your SSH keys

Retrieve all SSH keys registered to your account:

```bash
curl https://api.virtcloud.io/v1/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Deleting an SSH key

```bash
curl -X DELETE https://api.virtcloud.io/v1/ssh-keys/sk_abc123 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Note>
  Deleting an SSH key from your account removes it from VirtCloud's registry, but it does **not** remove the key from the `authorized_keys` file on any VM that was already provisioned with it. Those VMs remain accessible using the corresponding private key until you manually remove the entry from the VM itself.
</Note>

## Adding a key to an existing VM

If you need to grant access to a VM that was already created, inject an additional SSH key without reprovisioning:

```bash
curl -X POST https://api.virtcloud.io/v1/virtual-machines/vm_xyz789/ssh-keys \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "ssh_key_id": "sk_abc123"
  }'
```

VirtCloud appends the public key to the VM's `authorized_keys` file via the guest agent. The change takes effect within a few seconds — no reboot required.

## Security best practices

- **Use `ed25519` keys.** They are more secure and more efficient than RSA keys.
- **Use a different key per device.** Register one key for your laptop and a separate key for your desktop or CI environment. If one device is compromised, you can revoke only that key without losing access from other devices.
- **Protect your private key.** Set a passphrase when generating the key pair and store the private key only in your `~/.ssh` directory with permissions set to `600`.
- **Rotate keys periodically.** Add a new key, verify access, then delete the old one.
- **Remove stale keys.** Delete keys you no longer use from your VirtCloud account to reduce your attack surface.
